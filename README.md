# Java_Bonnes_Pratiques

1 - Clean code :

● Méthodes trop longues : Découpez les méthodes de plus de 30 lignes en sous-méthodes
cohérentes

● Nommage incohérent: Unifiez le nommage (français/anglais, conventions de nommage)

● Magic numbers : Extrayez les constantes en dur dans des variables nommées

● Manque de Javadoc/XML Documentation : Documentez les classes et méthodes publiques

● Code mort: Supprimez le code commenté ou inutilisé

● Indentation etformatage : Appliquez un formatage uniforme
